# Hanko - 電子署名ツール

公開鍵暗号を用いて電子署名の作成と検証を行うブラウザベースのツールです。鍵の生成、証明書の発行、文書への署名、そして検証までの完全なワークフローを提供します。

## デモ

メインアプリケーションでは、デジタルIDの作成と使用に必要なステップバイステップのワークフローを提供しています。

*   **[Hanko 電子署名ツール](https://code4fukui.github.io/hanko/)**

署名と検証の概念を素早く理解するには、以下の体験用チュートリアルをお試しください。

*   **[電子署名と検証の体験](https://code4fukui.github.io/hanko/experience.html)**

## 機能

*   **鍵生成**: ブラウザ上で直接、公開鍵と秘密鍵のペアを作成します。
*   **証明書発行**: 証明書発行元（CA）として機能し、公開鍵に署名して証明書を発行します。
*   **証明書管理**: 受け取った証明書と自身の秘密鍵を組み合わせ、完全なデジタルIDを作成します。
*   **文書署名**: 分離型の署名ファイル（`.hanko.txt`）を作成し、任意のファイルに電子署名します。
*   **署名検証**: 文書の署名が本物であること、および署名者の証明書が有効であることを検証します。
*   **インストール不要**: サーバーサイドのコンポーネントやインストールは一切不要で、すべてブラウザ上で動作します。

## ワークフロー

このツールは複数のステップに分かれており、それぞれ別のページで処理を行います。典型的なワークフローは、文書に署名したい**ユーザー**と、ユーザーの身元を検証する**証明書発行元（CA）**で構成されます。

#### 1. 鍵生成 (`init.html`)
**ユーザー**として、最初に鍵ペアを作成します。
1.  `init.html` を開きます。
2.  「公開鍵と秘密鍵を生成する」をクリックします。
3.  名前と強力なパスワードを入力します。
4.  以下の2つのファイルをダウンロードします。
    *   `name.seckey_no_cert.txt`: パスワードで保護された秘密鍵です。**これは誰にも教えず秘密に保管してください。**
    *   `name.pubkey.txt`: 公開鍵です。証明書を発行してもらうために証明書発行元（CA）へ送信します。

#### 2. 証明書発行 (`cert.html`)
**証明書発行元（CA）**として、ユーザーの公開鍵に署名します。
1.  `cert.html` を開きます。
2.  自身のCA秘密鍵ファイル（例: `ca.seckey.txt`）をドラッグ＆ドロップします。
3.  ユーザーの公開鍵ファイル（`name.pubkey.txt`）をドラッグ＆ドロップします。
4.  `name.certificate.txt` ファイルがダウンロードされます。この証明書ファイルをユーザーに返送します。

#### 3. 証明書登録 (`regist.html`)
**ユーザー**として、自身の秘密鍵と受け取った証明書を組み合わせます。
1.  `regist.html` を開きます。
2.  最初に生成した証明書なし秘密鍵ファイル（`name.seckey_no_cert.txt`）をドラッグ＆ドロップします。
3.  CAから受け取った証明書ファイル（`name.certificate.txt`）をドラッグ＆ドロップします。
4.  証明書付きの完全な秘密鍵ファイル（`name.seckey.txt`）がダウンロードされます。今後の署名にはすべてこのファイルを使用します。

#### 4. 電子署名 (`sign.html`)
**ユーザー**として、文書に署名できるようになります。
1.  `sign.html` を開きます。
2.  完全な秘密鍵ファイル（`name.seckey.txt`）をドラッグ＆ドロップし、パスワードを入力します。
3.  署名したいファイル（例: `document.pdf`）をドラッグ＆ドロップします。
4.  電子署名ファイル（`document.pdf.name.hanko.txt`）がダウンロードされます。

#### 5. 検証 (`verify.html`)
**誰でも**署名された文書を検証することができます。
1.  `verify.html` を開きます。
2.  元の署名されていないファイル（`document.pdf`）をドラッグ＆ドロップします。
3.  電子署名ファイル（`document.pdf.name.hanko.txt`）をドラッグ＆ドロップします。
4.  ツールが、その文書に対する署名が有効であるかを確認し、証明書の詳細を表示します。

## データ / API

このプロジェクトは以下のライブラリを使用して構築されています。
- [sec.js](https://github.com/code4fukui/sec.js): 暗号化操作
- [Base32](https://github.com/code4fukui/Base32): データエンコーディング
- [input-text](https://github.com/code4fukui/input-text): カスタムUIコンポーネント
- [qr-code](https://github.com/code4fukui/qr-code): QRコード生成
